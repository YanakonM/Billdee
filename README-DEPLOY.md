# คู่มือติดตั้ง Tex V2 ใช้งานจริงในโรงงาน (เครื่องเดียว)

## 1. Build และรันแบบ production

อย่าใช้ `npm run dev` ในงานจริง (ช้ากว่าและไม่เสถียรเท่า) ให้ build ครั้งเดียวแล้วรันตัว production:

```powershell
cd "C:\Users\yanak\Downloads\Tex V2"
npm run build        # สร้างไฟล์ production ลงโฟลเดอร์ dist
npm run preview      # เปิดเซิร์ฟเวอร์ที่ http://localhost:4173
```

> เปิดใช้ที่ **http://localhost:4173** (สังเกต: คนละพอร์ตกับตอน dev)

### ทำให้เปิดเองตอนเครื่องบูต (แนะนำ)
1. สร้างไฟล์ `start-texv2.bat` ไว้ที่ Desktop เนื้อหา:
   ```bat
   cd /d "C:\Users\yanak\Downloads\Tex V2"
   start "" "http://localhost:4173"
   npm run preview
   ```
2. กด `Win + R` พิมพ์ `shell:startup` → Enter → ลาก shortcut ของไฟล์ .bat ไปวาง
3. ทุกครั้งที่เปิดเครื่อง ระบบจะรันและเปิดหน้าเว็บให้เอง

## 2. ติดตั้งเป็นแอป (PWA)

1. เปิด http://localhost:4173 ด้วย **Chrome หรือ Edge**
2. กดไอคอน **Install** ที่มุมขวาของ address bar (รูปจอ+ลูกศร)
3. ได้ไอคอน Tex V2 บน Desktop — เปิดเหมือนโปรแกรมปกติ ไม่ต้องจำ URL

## 3. ความปลอดภัยของข้อมูล (สำคัญที่สุด)

ข้อมูลทั้งหมดอยู่ใน IndexedDB ของเบราว์เซอร์เครื่องนี้ **เครื่องเดียว** — ต้องมีสำรองเสมอ:

1. **ตั้งสำรองอัตโนมัติ**: ตั้งค่า → สำรอง/กู้คืน → "เลือกโฟลเดอร์สำรองอัตโนมัติ"
   - แนะนำเลือกโฟลเดอร์ใน **USB drive** หรือโฟลเดอร์ที่ **sync กับ Google Drive/OneDrive**
   - ระบบจะเขียนไฟล์สำรองให้เอง **วันละครั้ง** ตอนเปิดใช้งาน
   - หลังเปิดเบราว์เซอร์ใหม่ บางครั้ง Chrome จะขอให้กดยืนยันสิทธิ์ 1 คลิก — มีแบนเนอร์เตือนบนแดชบอร์ด
2. **ห้ามกด "Clear browsing data"** ใน Chrome เด็ดขาด — ข้อมูลจะหายทั้งหมด
   - แนะนำสร้าง Chrome profile แยกเฉพาะงานนี้ (คลิกรูปโปรไฟล์มุมขวาบน → Add)
3. กฎหมายไทยกำหนดเก็บเอกสารภาษีอย่างน้อย **5 ปี** — เก็บไฟล์สำรองไว้ 2 ที่เสมอ (USB + cloud)
4. ถ้าไฟโรงงานไม่นิ่ง ควรมี **UPS** ต่อกับ notebook

## 4. การตั้งค่าเครื่องพิมพ์ (ขนาดกระดาษ)

### ใบ A4 (ใบกำกับภาษี/ใบเสร็จเต็มรูป)
- ระบบสั่งขนาด A4 ให้อัตโนมัติ (`@page size: A4`)
- ในหน้าต่างพิมพ์ของ Chrome: Paper size = **A4**, Margins = **Default**
- เปิด **Background graphics** ถ้าหัวตารางสีเข้มไม่ออก (ระบบบังคับให้แล้ว แต่เผื่อไว้)

### ใบเสร็จย่อ 80mm (เครื่องพิมพ์ความร้อน/thermal)
- ติดตั้งไดรเวอร์เครื่องพิมพ์ก่อน แล้วตั้งขนาดกระดาษของไดรเวอร์เป็น **80mm (72.1mm printable)**
- ในหน้าต่างพิมพ์: เลือกเครื่องพิมพ์ thermal → Paper size เลือก **80 x 297 / Roll 80mm** → Margins = **None**
- ครั้งแรกให้พิมพ์ทดสอบ 1 ใบเทียบกับกระดาษจริง — ถ้าตัวอักษรล้น ให้เช็คว่าไม่ได้เลือก A4 ค้างไว้
- Chrome จะจำเครื่องพิมพ์+ขนาดล่าสุดที่ใช้ ทำให้ครั้งต่อไปกดพิมพ์ได้เลย

### เครื่องสแกนบาร์โค้ด
- ใช้สแกนเนอร์ USB แบบ keyboard-mode ได้ทันที — ยิงบาร์โค้ดใส่ช่อง "ค้นหาสินค้า" ในหน้าสร้างใบเสร็จ

## 5. กฎการใช้งานประจำวัน

- ใช้งาน **ทีละแท็บ** — ถ้าเปิดซ้อน ระบบจะเตือนและบล็อกแท็บที่สอง
- สิ้นวัน: ดูแดชบอร์ด ถ้ามีแบนเนอร์เตือนสำรองข้อมูล ให้กดสำรองก่อนปิดเครื่อง
- ทุกเดือน: ทดสอบไฟล์สำรองด้วยการเปิดดูว่าไฟล์ JSON มีขนาดสมเหตุสมผล (ไม่ใช่ 0 KB)

## 6. เข้าใช้จากคอมเครื่องอื่นใน LAN

มี 2 ชั้นที่ต้องเข้าใจ — เปิด "หน้าโปรแกรม" กับ "แชร์ข้อมูล" เป็นคนละเรื่อง

### ชั้นที่ 1: ให้คอมอื่นเปิดหน้าโปรแกรมได้
1. ที่เครื่องแม่ รัน server แบบเปิด LAN:
   ```powershell
   npm run preview -- --host
   ```
   Vite จะโชว์ Network URL เช่น `http://192.168.1.50:4173`
2. หา IP เครื่องแม่: `ipconfig` → ดู IPv4 Address
3. เปิด Windows Firewall ให้พอร์ต 4173 ผ่าน (ครั้งแรก Windows มักถามให้ Allow)
4. ที่คอมอื่น เปิด Chrome → `http://<IP เครื่องแม่>:4173`

### ⚠️ ชั้นที่ 2: ข้อมูลจะแชร์กันหรือไม่
- **โหมด IndexedDB (ค่าเริ่มต้น):** แต่ละคอมเก็บข้อมูลใน browser ของตัวเอง → **ข้อมูลไม่แชร์กัน** คอมอื่นจะเห็นว่างเปล่า เหมาะกับ "เครื่องเดียว" เท่านั้น
- **ต้องการให้ทุกคอมใช้ข้อมูลชุดเดียวกัน:** ต้องใช้ **PocketBase** → ดู [README-POCKETBASE.md](README-POCKETBASE.md)
  - เครื่องแม่รัน PocketBase
  - ทุกเครื่องสลับเป็นโหมด PocketBase แล้วชี้ URL ไป IP เครื่องแม่
  - ข้อมูลอยู่ที่ส่วนกลาง ทุกเครื่องเห็นเหมือนกัน

## 7. ย้ายเครื่อง / กู้คืนข้อมูล

1. เครื่องใหม่: ติดตั้ง Node.js → คัดลอกโฟลเดอร์โปรเจกต์ → `npm install` → `npm run build`
2. เปิดระบบ → ตั้งค่า → สำรอง/กู้คืน → **นำเข้า (Import)** → เลือกไฟล์สำรองล่าสุด
3. ตรวจเลขรันเอกสารใน ตั้งค่า → ใบเสร็จ ว่าตรงกับใบล่าสุดที่ออกไป
