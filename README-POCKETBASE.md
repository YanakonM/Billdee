# ใช้ PocketBase เก็บข้อมูล (เผื่อหลายเครื่องในโรงงาน)

ค่าเริ่มต้นของแอปคือเก็บข้อมูลในเครื่อง (IndexedDB) ใช้เครื่องเดียว ถ้าต้องการให้
**หลายเครื่องในโรงงานใช้ข้อมูลชุดเดียวกัน** ให้ตั้ง PocketBase เป็นศูนย์กลาง ตามขั้นตอนนี้

> ⚠️ แอปจะทำงานบน PocketBase ได้ก็ต่อเมื่อ **server รันอยู่** ถ้า server ดับ เครื่องลูกจะใช้ไม่ได้
> ดังนั้นเครื่องที่รัน PocketBase (เครื่องแม่) ควรเปิดทิ้งไว้และมี UPS

## 1. ติดตั้งและรัน PocketBase (ทำที่ "เครื่องแม่" เครื่องเดียว)

1. โหลด PocketBase (ไฟล์เดียว) จาก https://pocketbase.io/docs/ — เลือก Windows amd64
2. แตกไฟล์ได้ `pocketbase.exe` วางไว้ในโฟลเดอร์ เช่น `C:\TexPB\`
3. เปิด PowerShell ในโฟลเดอร์นั้น แล้วรัน (ใส่ `0.0.0.0` เพื่อให้เครื่องลูกใน LAN เข้าถึงได้):
   ```powershell
   .\pocketbase.exe serve --http=0.0.0.0:8090
   ```
4. ครั้งแรกจะมีลิงก์ให้ตั้ง **บัญชีผู้ดูแล (admin)** — เปิดลิงก์ในเบราว์เซอร์ ตั้งอีเมล/รหัสผ่าน
5. ข้อมูลทั้งหมดจะอยู่ในโฟลเดอร์ `pb_data/` ที่เกิดขึ้นข้างๆ — **โฟลเดอร์นี้คือฐานข้อมูลจริง สำรองมันไว้สม่ำเสมอ**

ให้ PocketBase เปิดเองตอนบูตเครื่องแม่: สร้าง shortcut ของคำสั่งข้างบนไว้ใน `shell:startup`
(หรือทำเป็น Windows Service ด้วย NSSM)

## 2. สร้าง Collections (7 อัน) ในหน้า Admin

ที่หน้า admin → **Collections → New collection** สร้างทีละอัน ชื่อ **ต้องตรงเป๊ะ**:

| Collection | ฟิลด์ที่ต้องเพิ่ม (นอกจากที่ระบบมีให้) |
|-----------|------------------------------------------|
| `customers` | `data` (type **JSON**), `name` (Text), `code` (Text) |
| `products` | `data` (JSON), `barcode` (Text), `code` (Text) |
| `invoices` | `data` (JSON) |
| `quotations` | `data` (JSON) |
| `creditNotes` | `data` (JSON) |
| `stockLogs` | `data` (JSON) |
| `settings` | `data` (JSON), `key` (Text) → ตั้ง **Unique** ที่ฟิลด์ key |

> ชื่อ collection ต้องเป็น `creditNotes` และ `stockLogs` (ตัวพิมพ์ใหญ่ตรงกลาง) ตามนี้

## 3. ตั้งสิทธิ์เข้าถึง (API Rules)

แอปนี้คุยกับ PocketBase แบบไม่มี login (เหมาะกับ LAN ปิดในโรงงาน) — แต่ละ collection
ไปที่แท็บ **API Rules** แล้วตั้งทั้ง 5 rule (List/View/Create/Update/Delete) ให้ **เปิด**
โดยปล่อยช่องว่าง (เครื่องหมาย ✓ สีเขียว = public)

> 🔒 ความปลอดภัย: ทำแบบนี้ได้เฉพาะเครือข่ายภายในโรงงานที่ไม่เปิดออกอินเทอร์เน็ต
> อย่า forward port 8090 ออกเน็ต ถ้าต้องการความปลอดภัยกว่านี้ค่อยเพิ่มระบบ login ทีหลัง (บอกผมได้)

## 4. สลับแอปมาใช้ PocketBase + ย้ายข้อมูล

1. **ก่อนสลับ** (ยังอยู่โหมด IndexedDB): ตั้งค่า → สำรอง/กู้คืน → **ส่งออก (Export)** เก็บไฟล์ JSON ไว้
2. ตั้งค่า → **ที่เก็บข้อมูล** → เลือก *PocketBase server* → ใส่ URL
   - เครื่องแม่เอง: `http://127.0.0.1:8090`
   - เครื่องลูก: `http://<IP เครื่องแม่>:8090` เช่น `http://192.168.1.50:8090`
   - กด **ทดสอบการเชื่อมต่อ** ให้ขึ้น ✅ ก่อน
3. กด **บันทึกและรีโหลด**
4. หลังรีโหลด: ตั้งค่า → สำรอง/กู้คืน → **นำเข้า (Import)** เลือกไฟล์ JSON ที่ส่งออกไว้ในข้อ 1
   → ข้อมูลทั้งหมดจะถูกอัปขึ้น PocketBase
5. เสร็จ — เครื่องนี้ทำงานบน PocketBase แล้ว

**เครื่องลูกเครื่องอื่น:** ทำแค่ข้อ 2–3 (ใส่ IP เครื่องแม่) ไม่ต้อง import ซ้ำ เพราะข้อมูลอยู่บน server แล้ว

## 5. การสำรองข้อมูลเมื่อใช้ PocketBase

- สำรองโฟลเดอร์ `pb_data/` ของเครื่องแม่สม่ำเสมอ (คือฐานข้อมูลจริงทั้งหมด) — copy ขึ้น USB/cloud
- ปุ่ม **ส่งออก (Export)** ในแอปก็ยังใช้ได้ (ดึงจาก PocketBase มาเป็นไฟล์ JSON)
- หากเครื่องลูกเปิดอยู่ตอน server ดับ จะใช้งานไม่ได้จนกว่า server กลับมา

## 6. กลับไปใช้ IndexedDB

ตั้งค่า → ที่เก็บข้อมูล → เลือก *ในเครื่อง (IndexedDB)* → บันทึกและรีโหลด
(ข้อมูลใน IndexedDB เดิมยังอยู่ ไม่ได้ถูกลบตอนสลับไป PocketBase)

## หมายเหตุทางเทคนิค

- เลขรันเอกสาร (INV/QT/CN): บนเครื่องเดียวทำงานถูกต้อง หากมีหลายเครื่องออกบิล**พร้อมกันเป๊ะๆ**
  บ่อยมาก อาจต้องเพิ่ม server-side hook เพื่อจองเลขแบบ atomic ข้ามเครื่อง — แจ้งผมถ้าต้องการ
- โครงสร้างเก็บข้อมูลแบบ `data` (JSON) + คอลัมน์ index ไม่กี่ตัว ทำให้เพิ่มฟิลด์ใหม่ในอนาคต
  ไม่ต้องแก้ schema ใน PocketBase
